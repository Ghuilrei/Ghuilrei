Java异常体系

Posted on 2021-07-12 Edited on 2021-07-26 In 语言基础

Java中，异常对象都是派生于Throwable类的一个实例。

异常分类

Error类

Error类层次结构描述了 Java 运行时系统的内部错误和资源耗尽错误。

应用程序不应该抛出这种类型的对象。

如果出现了这样的内部错误，除了通告给用户，并尽力使程序安全地终止之外， 再也无能为力了。

Exception类

  • RuntimeException
  • 其他异常

由程序错误导致的异常属于RuntimeException。

而程序本身没有问题，但由于像I/O错误这类问题导致的异常属于其他异常。

  • 派生于 RuntimeException 的异常
    • 错误的类型转换。
    • 数组访问越界 i
    • 访问 null 指针
  • 不是派生于RuntimeException的异常包括:
    • 试图在文件尾部后面读取数据。
    • 试图打开一个不存在的文件。
    • 试图根据给定的字符串查找 Class 对象， 而这个字符串表示的类并不存在,

如果出现RuntimeException异常那么就一定是你的问题

受查异常

Java将派生于Error和RuntimeException的异常成为非受查异常

其他异常称为受查异常

强烈建议解搞合 try/catch 和 try/finally 语句块。 这样可以提高代码的清晰度。

使用技巧

  1. 异常处理不能代替简单的测试，只在异常情况下使用异常机制。
  2. 不要过分地细化异常。
  3. 利用异常层次结构。
    1. 不要只抛出 RuntimeException 异常。应该寻找更加适当的子类或创建自己的异常类。
    2. 不要只捕获 Thowable 异常， 否则， 会使程序代码更难读、更难维护。
  4. 不要压制异常
  5. 在检测错误时，“苛刻”要比放任更好
  6. 不要羞于传递异常

try catch finally 中的 return 问题

情况1

return在最外面

执行顺序：

程序按顺序执行。

情况2

try语句中有return

执行顺序：

  1. 执行try块中return之前代码（包括return语句中的表达式运算）
  2. 执行finally块
  3. 最后执行try中return;

finally块之后的语句return，因为程序在try中已经return所以不再执行。

情况3

执行顺序：

  1. 程序先执行try

    • 有异常

      1. 执行catch中return之前代码（包括return语句中的表达式运算）
      2. 执行finally语句中全部代码
      3. 最后执行catch块中return

      finally之后也就是4处的代码不再执行。

    • 无异常：

      1. 跳过catch块顺序执行

情况4：

执行顺序：

  1. 执行try块中return之前代码（包括return语句中的表达式运算）
  2. 执行finally块，因为finally块中有return所以提前退出。

情况5

执行顺序：

  1. 程序执行catch块中return之前代码（包括return语句中的表达式运算）
  2. 再执行finally块，因为finally块中有return所以提前退出。

情况6

执行顺序：

  1. 程序执行try块中return之前代码（包括return语句中的表达式运算）
    • 有异常：
      1. 执行catch块中return之前代码（包括return语句中的表达式运算）
      2. 则再执行finally块，因为finally块中有return所以提前退出。
    • 无异常：
      1. 则再执行finally块，因为finally块中有return所以提前退出。

结论：

  1. 任何执行try或者catch中的return语句之前，都会先执行finally语句，如果finally存在的话。
  2. 如果finally中有return语句，那么程序就return了，所以finally中的return是一定会被return的，
  3. finally是在return的表达式运算后执行的，此时并没有直接返回运算后的值，而是先把要返回的值保存起来，但是不管finally中的代码怎么样，返回的值都不会改变，仍然是之前保存的值，所以函数返回值是在finally执行前确定的；
  4. finally中最好不要包含return，否则程序会提前退出，返回值不是try或catch中保存的返回值，编译器把finally中的return实现为一个warning。
